# 功能规格：从现有组件中沉淀设计风格指南

**功能分支**：`20260329-distill-design-style`
**创建时间**：2026-03-29
**状态**：草稿
**输入**：用户描述："基于目前的组件，沉淀出我的设计风格"
**语言要求**：本文件及同级 `/specs/` 文档正文必须使用简体中文；代码标识符、命令、文件路径、协议字段和第三方 API 名称可保留原文。

---

## 背景

通过系统性地分析现有 Button、Popover、Text 组件，以及 `@deweyou-ui/styles` 中的设计 token，将隐含在代码中的设计决策显式化为可参考、可传承的设计风格文档。目标是让后续新增的任何组件，都能在无需反复查阅现有实现的情况下，保持一致的视觉与交互风格。

---

## 用户场景与测试（必填）

### 用户故事 1 - 设计师查阅风格规则（优先级：P1）

设计师在新建组件时，能通过本规格文档快速了解调色板、字体、圆角、间距、动效等核心设计决策，而无需逆向阅读现有组件代码。

**为什么是这个优先级**：这是本功能的核心价值——将隐性知识显式化，直接降低新组件的设计偏差风险。

**独立测试**：可以将规格文档单独交给一位未参与本项目的设计师，验证其能否仅凭文档重现与现有组件一致的视觉稿。

**验收场景**：

1. **假如**设计师需要为新组件选择品牌色，**当**查阅本规格，**那么**能找到明确的 emerald 绿色品牌主色与语义色（primary、danger、neutral）定义，以及每种颜色对应的用途说明。
2. **假如**设计师需要确定新浮层的圆角和阴影，**当**查阅本规格，**那么**能找到统一的浮层圆角（0.4rem）和柔和阴影规则，以及何时使用 rect / rounded 的判断准则。

---

### 用户故事 2 - 开发者实现新组件时对齐风格（优先级：P1）

开发者在实现新交互型组件（如 Dialog、Menu、Tooltip）时，能直接依照本规格确定 token 用法、动效参数、焦点样式、交互反馈，而不必从 Button 或 Popover 源码中抽取规律。

**为什么是这个优先级**：开发者是风格一致性的直接执行方，规格缺失会导致每个组件各自为政。

**独立测试**：可以仅凭本规格文档，实现一个风格与现有组件完全一致的简单组件（如 Tag），无需查看其他组件源码。

**验收场景**：

1. **假如**开发者需要为新组件添加 hover/active 交互动效，**当**查阅本规格，**那么**能找到统一的过渡时长（140ms ease）和按压微动效（translateY(1px)）规则。
2. **假如**开发者需要为浮层组件添加出入场动画，**当**查阅本规格，**那么**能找到统一的动画时长（160ms）、缓动函数（入场 cubic-bezier(0.22, 1, 0.36, 1)，出场 ease）和位移比例（6px + scale(0.98)）。
3. **假如**开发者需要处理焦点样式，**当**查阅本规格，**那么**能找到统一的焦点环规则（2px emerald outline，2px offset）。

---

### 用户故事 3 - 审查新组件是否符合设计风格（优先级：P2）

在 Code Review 或设计走查时，审查者能用本规格作为检查清单，快速判断新组件是否偏离了既定风格。

**为什么是这个优先级**：规格文档的长效价值在于可被持续引用，作为合规检查依据。

**独立测试**：可以将一个故意引入风格偏差的组件提交给审查者，验证其能否仅凭本规格定位并描述偏差所在。

**验收场景**：

1. **假如**新组件的 disabled 状态使用了 `opacity: 0.3`，**当**审查者对照本规格，**那么**能指出应使用 `opacity: 0.56` 并注明出处。
2. **假如**新组件的动画时长使用了 `300ms`，**当**审查者对照本规格，**那么**能指出交互反馈应为 140ms，浮层动画应为 160ms。

---

### 边界情况

- 当某个新组件的视觉需求无法完全套用已有规则时（如超大尺寸展示型组件），风格规格应明确说明可扩展点，而不是过度约束。
- 当 dark 主题下的 token 值与 light 主题不同时，风格规格应说明主题切换机制，而不是为每个 token 列举双份值。
- 仅使用键盘的用户操作任何组件时，焦点环规则必须始终适用，不受主题影响。

---

## 需求（必填）

### 功能需求

- **FR-001**：风格文档必须覆盖**字体排印系统**，包含字体族（宋体优先）、字重梯度（400/500/600/700）、字号层级（caption/body/h1-h5）和行高规则。
- **FR-002**：风格文档必须覆盖**色彩系统**，包含品牌主色（emerald 绿）、语义色（primary、neutral、danger）、画布/表面/边框/文字的层级关系，以及 token 命名规范。
- **FR-003**：风格文档必须覆盖**形状与圆角系统**，包含 rect / rounded / pill 三种圆角档位的适用场景，以及浮层与交互元素的默认值。
- **FR-004**：风格文档必须覆盖**尺寸与间距系统**，包含按钮的五级尺寸比例（xs/sm/md/lg/xl）和组件内边距的比例关系。
- **FR-005**：风格文档必须覆盖**阴影系统**，包含柔和阴影（`--ui-shadow-soft`）的视觉意图与适用场景（浮层、卡片等）。
- **FR-006**：风格文档必须覆盖**交互反馈规则**，包含 hover（6%/12% 透明混色）、active（10%/18% 透明混色 + 1px 向下位移）、disabled（opacity 0.56，禁止点击）、loading（spinner 覆盖图标）的统一规则。
- **FR-007**：风格文档必须覆盖**过渡与动画系统**，包含交互元素过渡（140ms ease）、浮层入场（160ms cubic-bezier(0.22, 1, 0.36, 1)）、浮层出场（160ms ease forwards）、以及 `prefers-reduced-motion` 降级规则。
- **FR-008**：风格文档必须覆盖**焦点与无障碍规则**，包含焦点环（2px emerald outline，2px offset）、keyboard navigation 约定、ARIA 属性使用规范（aria-disabled vs disabled、aria-busy 等）。
- **FR-009**：风格文档必须覆盖**组件变体模型**，说明 variant（视觉层级：filled/outlined/ghost/link）、color（语义色：neutral/primary/danger）、size、shape 四个维度的正交关系与约束。
- **FR-010**：风格文档必须说明**主题机制**，包含 CSS 自定义属性作为 token 载体的约定，以及通过 `[data-theme]` 属性切换 light/dark 的实现模式。

### 无障碍与 UI 契约（UI 工作必填）

- **用户角色 / 参与方**：消费 `@deweyou-ui/components` 的应用开发者、维护本设计系统的设计师、进行 Code Review 的工程师
- **交互模型**：本文档为静态规格文档，不含可交互组件；但所记录的规则覆盖键盘导航、屏幕阅读器标签、焦点管理
- **需要覆盖的状态**：默认、hover、focus-visible、disabled、loading、active（按压）
- **主题 / token 影响**：仅整理现有 token，不新增；light/dark 主题切换机制需在文档中说明

---

## 成功标准（必填）

### 可度量结果

- **SC-001**：完成文档后，开发者实现一个新组件（如 Tag）时，因风格问题需要查阅现有组件源码的次数为 0。
- **SC-002**：设计走查中，因风格文档缺失导致的偏差问题，与文档完成前相比减少 80% 以上。
- **SC-003**：本规格文档覆盖现有组件中所有可通过 CSS token 或 prop 参数化的设计决策，覆盖率达到 100%（以 Button、Popover、Text 三个现有组件为基准）。
- **SC-004**：规格中每条规则均可追溯到至少一个现有组件的实现作为参考佐证。
- **SC-005**：所有规则以非技术语言描述设计意图，同时提供对应的 token 名称或具体数值，使设计师和开发者均可直接使用。

### 品牌色与语义色 token 对照

| 语义         | Token 名称                 | 调色板值    |
| ------------ | -------------------------- | ----------- |
| 品牌/主色    | `--ui-color-brand-bg`      | emerald-700 |
| 危险操作     | `--ui-color-danger-bg`     | red-700     |
| 焦点环       | `--ui-color-focus-ring`    | emerald-500 |
| 链接         | `--ui-color-link`          | emerald-700 |
| 正文文字     | `--ui-color-text`          | neutral-950 |
| 画布（底层） | `--ui-color-canvas`        | white       |
| 表面（卡片） | `--ui-color-surface`       | neutral-50  |
| 边框（默认） | `--ui-color-border`        | slate-300   |
| 边框（强调） | `--ui-color-border-strong` | slate-400   |

### 圆角档位

| 档位    | 值     | 适用场景                        |
| ------- | ------ | ------------------------------- |
| rect    | 0      | 贴合容器边缘的嵌入式元素        |
| rounded | 0.4rem | ghost/link 按钮、浮层、小型控件 |
| auto    | 0.8rem | filled/outlined 按钮默认        |
| pill    | 999px  | 标签、胶囊型按钮                |

### 交互状态色彩公式

```
hover 背景  = color-mix(in srgb, <base-color>  6–12%, transparent)
active 背景 = color-mix(in srgb, <base-color> 10–18%, transparent)
disabled    = opacity: 0.56，cursor: not-allowed
```

### 动效参数汇总

```
交互元素过渡：140ms ease（属性：background, border-color, color, box-shadow, transform）
按压微动效：  translateY(1px)
浮层入场：    160ms cubic-bezier(0.22, 1, 0.36, 1)
              from: translateY/X(6px) + scale(0.98) + opacity 0
              to:   translate3d(0,0,0) + scale(1) + opacity 1
浮层出场：    160ms ease forwards
              from: translate3d(0,0,0) + scale(1) + opacity 1
              to:   translateY/X(4.2px) + scale(0.98) + opacity 0
link 下划线： 260ms ease（clip-path 展开动画）
loading 旋转：0.9s linear infinite
prefers-reduced-motion：禁用 clip-path 过渡；loading 动画延长至 1.8s；浮层动效归零
```
