# 归档：Tabs 组件

**分支**：`20260331-tabs-component`  
**完成时间**：2026-03-31  
**类型**：feat

---

## 交付摘要

基于 `@ark-ui/react` Tabs 原语实现了功能完整的 Tabs 组件，支持横排/竖排（`orientation`）、自动/手动激活（`activationMode`）、受控/非受控模式。提供两种激活指示器变体：`line`（横排底部下划线/竖排左侧竖线，带平滑切换动画 160ms）和 `bg`（背景填充）。Tab 项可承接 Menu 下拉子菜单（横排向下/竖排向右展开），支持 `hideContent` 仅标签栏模式。超长处理提供 `scroll`（渐变遮罩滚动）和 `collapse`（溢出收入"更多"下拉菜单）两种策略，`prefers-reduced-motion` 时指示器动画自动禁用。

## 关键决策

| 决策                      | 选择                                                      | 理由                                               | 备选方案          |
| ------------------------- | --------------------------------------------------------- | -------------------------------------------------- | ----------------- |
| 指示器动画实现            | CSS `transform` + `transition` 动态计算目标 tab 位置      | 流畅的物理感滑动，比直接切换类名更自然             | 切换 CSS 类名跳切 |
| 超长默认策略              | `scroll`（渐变遮罩）                                      | 实现相对简单，无需测量溢出宽度                     | `collapse`        |
| `collapse` 指示器边界处理 | 激活 tab 切至溢出区域时指示器滑向"更多"按钮方向并收缩至零 | 避免 `display:none` 的 `offsetLeft=0` 导致方向错误 | 直接隐藏指示器    |
| Tab 子菜单复用            | 复用 Menu/Popover 的弹出定位机制                          | 不重复实现定位逻辑                                 | 自行实现下拉定位  |

---

## 踩坑记录

- **问题**：`collapse` 模式下，溢出 tab 的 `display:none` 导致 `offsetLeft=0`，指示器动画方向计算出错（向左跳至原点）。  
  **解决方案**：动画位置计算时排除 `display:none` 的溢出 tab，仅对可见 tab 计算 `offsetLeft`，切至溢出区域时向"更多"按钮方向收缩。

---

## 可复用模式

- [指示器物理感滑动]：通过计算目标 tab 的 `offsetLeft`/`offsetTop` 动态设置指示器 `transform`，实现类原生的标签切换滑动感，后续类似选择器组件可复用此方案。
