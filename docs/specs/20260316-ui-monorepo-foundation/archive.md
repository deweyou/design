# 归档：UI Monorepo 基础建设

**分支**：`20260316-ui-monorepo-foundation`  
**完成时间**：2026-03-16  
**类型**：build

---

## 交付摘要

建立了基于 Vite+ 的 React + TypeScript + Less UI 组件库 monorepo 架构，明确划分 `packages/`（utils、hooks、styles、components）与 `apps/`（website、storybook）的职责边界。引入受控主题系统，以 TypeScript token 为源输出 CSS 主题文件，组件样式采用 CSS Modules 并要求显式导入全局样式，公开主题面仅暴露品牌色等少量 token，其余内部 token 不作为消费契约。

---

## 关键决策

| 决策                 | 选择                               | 理由                                         | 备选方案            |
| -------------------- | ---------------------------------- | -------------------------------------------- | ------------------- |
| 全局样式导入方式     | 消费者显式导入                     | 避免组件静默注入全局样式，保持透明性         | 组件内部自动注入    |
| 公开主题面范围       | 仅品牌色等少量 token               | 库有明确设计立场，不打开整套设计语言重塑能力 | 暴露完整 token 体系 |
| 组件本地样式         | CSS Modules + `index.module.less`  | 默认作用域隔离，防止样式泄漏                 | 全局 BEM 命名       |
| website vs Storybook | 职责分离：公开文档 vs 内部状态验证 | 避免示例重复维护，职责清晰                   | 合并为单一文档站    |

## 可复用模式

- [显式样式导入契约]：全局样式必须由消费者显式导入，组件包不静默注入，降低副作用风险。
- [受控主题面]：将公开主题 token 限定在品牌色等最小集合，其余 token 视为内部实现，控制可定制边界。
