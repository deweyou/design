# 契约：样式与主题

## 目的

定义面向消费者的公开样式契约，以及 `styles` 与 `components` 之间的内部样式边界。

## 消费者接入契约

- 消费者在依赖组件表现之前，必须显式导入文档中说明的全局样式入口。
- 文档化的接入路径必须在不要求任何自定义主题覆盖的前提下提供完整默认设计语言。
- 消费者可通过文档中说明的主题机制启用亮色或暗色主题行为。

## 主题产物契约

- styles package 必须提供：
  - 独立的亮色主题输出
  - 独立的暗色主题输出
  - 一个适合标准消费者接入的、文档化的默认主题入口
- 主题产物必须支持运行时在亮色 / 暗色模式间切换。

## 公开主题面契约

- 初始公开主题面必须刻意限制在关键颜色 token 上。
- 公开主题 tokens 至少包括：
  - 品牌背景
  - 品牌 hover 状态
  - 品牌 active 状态
  - 品牌背景上的文字色
  - 焦点强调
  - 链接强调
- 公开主题 tokens 用于品牌适配，而不是用于结构性重设计。

## 内部 Token 契约

- 结构、间距、圆角、字号体系、动效以及其他设计语言 token 默认都保持内部私有。
- 内部 tokens 定义组件库的视觉身份，不属于初始消费者自定义承诺的一部分。

## 组件样式契约

- 组件本地样式按组件实现，并与全局主题层和 reset 层保持分离。
- 组件本地样式必须使用 CSS Modules。
- 内部 CSS Module 类名属于实现细节，不是公开自定义钩子。
- 每个组件都必须支持根级 `className` 作为默认公开覆盖入口。

## 局部覆盖契约

- 组件可以暴露一组有限的组件级 CSS 变量，用于有价值的局部视觉调整。
- 初始组件级 CSS 变量应优先覆盖颜色相关调整。
- 在 v1 中，组件级覆盖不得开放对布局、间距、圆角或动效的无限制重塑能力。
