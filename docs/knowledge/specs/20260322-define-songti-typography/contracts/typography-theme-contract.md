# 契约：`@deweyou-ui/styles` 全局字体系统

## Package 目的

`@deweyou-ui/styles` 负责提供 Deweyou UI 的默认全局字体契约，包括默认宋体方向、平台回退规则、字重层级入口和例外字体角色。`apps/storybook` 与 `apps/website` 只负责评审和展示，不定义新的可复用字体规则。

## 受影响 package 与对外影响

- `@deweyou-ui/styles`
  - 不新增新的包导出路径。
  - 现有 `theme.css`、`theme-light.css`、`theme-dark.css`、`base.css` 中默认字体栈会从当前无衬线方向切换到宋体方向。
  - `--ui-font-body` 与 `--ui-font-display` 继续存在，但默认值更新为思源宋体系方向。
  - `--ui-font-mono` 继续作为等宽例外角色存在。
  - 这属于默认视觉契约更新，需要发布说明，但不新增新的公开 API。
- `@deweyou-ui/components`
  - 不新增 props、slots、events 或 variants。
  - 组件继续消费 `@deweyou-ui/styles` 提供的现有字体角色，因此会被动获得新的默认字体效果。
- `apps/storybook`、`apps/website`
  - 仅更新内部评审与公开指导内容，无对外 semver 影响。

## 字体角色契约

| 角色                | 默认方向 | 默认来源     | 平台回退                                                   | 例外说明                                           |
| ------------------- | -------- | ------------ | ---------------------------------------------------------- | -------------------------------------------------- |
| `--ui-font-body`    | 宋体     | 思源宋体系   | `macOS -> Songti SC, STSong`；`Windows -> SimSun, NSimSun` | 用于正文、控件、表单和常规数据文本                 |
| `--ui-font-display` | 宋体     | 思源宋体系   | `macOS -> Songti SC, STSong`；`Windows -> SimSun, NSimSun` | 用于标题与更强层级文本；不再保留几何无衬线默认方向 |
| `--ui-font-mono`    | 等宽     | 现有 mono 栈 | 现有 mono 回退                                             | 仅用于代码、固定宽度编号和文档化例外               |

## 默认字体与回退契约

- 默认开源字体目标必须锁定为思源宋体系，接受 `Source Han Serif SC` 与 `Noto Serif CJK SC` 这类同源发行。
- 默认开源字体未就绪、不可用或加载失败时，文本必须先使用系统回退字体显示，不得出现等待字体导致的空白文本。
- 平台回退必须明确锁定为：
  - `macOS`：`Songti SC`、`STSong`
  - `Windows`：`SimSun`、`NSimSun`
- 正文和标题都必须沿用同一宋体方向；不得让标题继续使用与正文明显割裂的默认无衬线家族。

## 字重层级契约

- 字重必须至少覆盖四个语义层级：正文、次强调、标题、强强调。
- 默认开源字体应优先提供完整层级。
- 当平台回退字体可用权重不足时，可以退到最接近可用值，但预览和评审必须仍能看出稳定层级。
- 不得将“只有 normal / bold 两档”视为本期最终目标。

## 中西文与数字混排契约

- 英文和数字默认跟随思源宋体系内置字形，不额外配置第二套西文字体。
- 默认混排规则必须覆盖中文、英文、数字、常见标点、日期、价格、百分比和版本号。
- 默认方案下，混排示例不得依赖单场景额外字体切换才能成立。

## 无障碍与可读性契约

- 字体调整不得削弱焦点可见性、语义结构或辅助技术可理解性。
- 紧凑按钮、表单控件、状态标签和数据密集文本必须在小尺寸下保持可读。
- 代码与固定宽度编号必须继续使用等宽例外角色，而不是强制并入宋体方向。
- 在字体未就绪和系统回退阶段，文本仍必须保持可读和层级可辨。

## 预览与验证契约

- `apps/storybook` 必须提供覆盖标题、正文、按钮、表单、数据展示和等宽例外文本的内部评审示例。
- `apps/website` 必须提供面向消费者的字体方向说明和中西文混排展示。
- Storybook 与 website 都必须至少包含一组中文、英文、数字同屏的混排样例。
- 最低验证集必须包含：
  - `vp check`
  - `vp test`
  - `vp run storybook#build`
  - `vp run website#build`
