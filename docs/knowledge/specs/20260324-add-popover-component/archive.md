# 归档：Popover 组件

**分支**：`20260324-add-popover-component`  
**完成时间**：2026-03-24  
**类型**：feat

---

## 交付摘要

新增 `Popover` 组件，支持 `click`/`hover`/`context-menu`/`focus` 四种触发方式（默认 `click`）及多触发方式组合，提供上/下/左/右等 8 个展示位置并在空间不足时自动回退。支持 `defaultVisible`/`visible`/`onVisibleChange` 构成的标准受控/非受控开关契约，`content` 必填，面板内部点击默认保持打开。视觉上提供 border + shadow + 指向箭头，支持 `rect`/`rounded` 两档圆角和 `card`/`loose`/`pure` 三种内容样式。新增 `boundaryPadding` 和 `popupPortalContainer` 定位约束。

---

## 关键决策

| 决策             | 选择                 | 理由                                        | 备选方案       |
| ---------------- | -------------------- | ------------------------------------------- | -------------- |
| 默认触发方式     | `click`              | 最常见场景，避免 hover 误触                 | `hover`        |
| 面板内部点击行为 | 默认保持打开         | 面板内常有可交互内容，不应被内部点击关闭    | 内部点击也关闭 |
| 多 Popover 互斥  | 默认不互斥           | 每个 Popover 独立管理状态，互斥由业务层编排 | 默认全局互斥   |
| 键盘焦点策略     | 非模态，不做焦点陷阱 | 保持页面其余内容可访问，键盘可进出内容区    | 模态焦点陷阱   |

## 可复用模式

- [受控/非受控开关契约]：`defaultVisible`/`visible`/`onVisibleChange` 三者构成的标准受控/非受控模式，后续所有浮层类组件（Menu、Tooltip 等）复用此模式。
- [非模态浮层键盘策略]：打开不做焦点陷阱，键盘可进入内容区和离开，关闭后焦点归还触发元素，是本仓库浮层组件的默认无障碍约定。
