# 组件入口契约

## 适用范围

- `@deweyou-ui/components`
- 当前已公开组件：`button`、`popover`、`text`
- 未来新增公开组件沿用同一契约

## 消费面

### 根入口

- 消费路径：`@deweyou-ui/components`
- 责任：保留现有聚合导出，兼容已存在消费者

### 子路径入口

- 消费路径：`@deweyou-ui/components/<component-name>`
- 责任：为每个公开组件提供稳定、可独立导入的入口

## 契约规则

1. 每个公开组件都必须同时具备根入口可达性和子路径可达性。
2. 使用子路径入口时，消费者不得被要求额外导入组件专属样式入口。
3. 子路径入口必须与根入口暴露相同的公开组件行为和无障碍表现。
4. 根入口继续作为兼容入口保留，不能因为新增子路径入口而失效。
5. 新增、移除或重命名任一子路径入口，都必须按 semver 评估影响。

## 验收信号

- 通过子路径入口导入任一公开组件时，可直接渲染且样式完整。
- 通过根入口导入同一组件时，现有行为保持不变。
- `apps/website` 或 `apps/storybook` 中至少存在一个验证单组件入口的示例。

## 破坏性变更判定

以下任一情况视为 breaking change：

- 移除已有子路径入口
- 修改已有子路径入口名称
- 新版本要求消费者补充新的组件专属样式导入步骤
- 子路径入口与根入口出现不一致的公开行为
