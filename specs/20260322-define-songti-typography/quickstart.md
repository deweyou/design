# 快速开始：验证组件库宋体字体系统

## 1. 安装与准备

1. 在仓库根目录执行 `vp install`。
2. 确认当前分支为 `20260322-define-songti-typography`。
3. 确认本地环境满足 Node.js `>=24.14.0`。

## 2. 本地预览

1. 执行 `vp run storybook#dev`，打开内部评审面。
2. 执行 `vp run website#dev`，打开公开指导面。
3. 在两个预览面中分别检查以下内容：
   - 标题、正文、按钮、表单和数据展示都已切换到统一宋体方向。
   - 中文、英文和数字混排时不需要额外西文字体即可保持协调。
   - 代码或固定宽度编号仍保持等宽例外表现。
   - light / dark 两种主题下字体方向一致。

## 3. 回退与边界检查

1. 观察默认字体尚未可用或被禁用时，文本是否先以系统回退字体显示。
2. 在 `macOS` 上确认回退符合 `Songti SC` / `STSong` 方向。
3. 在 `Windows` 上确认回退符合 `SimSun` / `NSimSun` 方向。
4. 检查正文、次强调、标题和强强调四个层级在默认字体与平台回退下都能被区分。
5. 检查紧凑控件、小字号标签和数据密集文本在回退状态下仍可读。

## 4. 自动化验证

1. 执行 `vp check`。
2. 执行 `vp test`。
3. 执行 `vp run storybook#build`。
4. 执行 `vp run website#build`。

## 5. 评审通过标准

- `@deweyou-ui/styles` 继续只暴露既有字体角色，不新增公开 typography token surface。
- 默认字体目标收敛到思源宋体系。
- 英文和数字默认跟随同一体系内置字形。
- 预览面已覆盖标题、正文、控件、数据展示和等宽例外。
- 默认字体未就绪时没有出现空白文本或明显不协调的回退风格。
