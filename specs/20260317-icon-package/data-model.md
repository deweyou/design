# 数据模型：为 UI 组件库新增图标包

## 实体：图标目录项

- **目的**：表示共享目录中的一个官方受支持图标。
- **字段**：
  - `name`：用于泛型 `Icon` API 的稳定消费者名称
  - `exportName`：映射到同一图标的稳定命名导出标识符
  - `category`：如 action、status、navigation、feedback 等基础分类
  - `sourceCollection`：用于保持目录视觉一致性的内部图标来源家族
  - `sourceKey`：该图标在上游来源中的内部标识符
  - `keywords`：供贡献者在文档和评审面中搜索发现的关键词
  - `directional`：该图标是否具有 left/right 或 start/end 等方向语义，需要额外评审
  - `status`：`active`、`deprecated` 或 `removed`
  - `replacementName`：当条目废弃时，可选的替代名称
  - `a11yGuidance`：描述该图标在何时通常应视为装饰性、何时应视为有意义图标的指导说明
- **校验规则**：
  - `name` 在官方目录中必须唯一。
  - `exportName` 在 package 公开 API 中必须唯一。
  - `category` 必须落在 spec 规定的基础目录范围内。
  - `status=deprecated` 时必须提供迁移说明。
  - `status=removed` 时不得继续通过公开 registry 可用。

## 实体：图标定义

- **目的**：表示 package 内部用于渲染的矢量图数据。
- **字段**：
  - `viewBox`：渲染时使用的坐标系元数据
  - `body`：渲染该图标所需的路径或形状数据
  - `defaultStrokeOrFill`：用于保持视觉家族一致性的来源元数据
  - `intrinsicAlignment`：在需要时用于视觉光学校正的可选元数据
- **关系**：
  - 每个 `图标目录项` 精确引用一个活跃的 `图标定义`。
  - 一个来源家族可以提供多个 `图标定义`。
- **校验规则**：
  - 同一公开目录中的图标定义，渲染后必须与选定视觉家族保持一致。
  - 图标定义必须在无网络查找的情况下完成渲染。

## 实体：图标渲染请求

- **目的**：表示消费者通过 package 请求渲染一个图标的行为。
- **字段**：
  - `name`：使用泛型入口时传入的官方图标名
  - `size`：受支持的 package 尺寸值或消费者显式覆盖值
  - `label`：可选的可访问名称；若省略，则图标视为装饰性
  - `className`：公开样式钩子
  - `tone`：可选语义颜色角色，默认继承周围 UI 样式
- **校验规则**：
  - `name` 必须解析到一个活跃目录项，否则请求应显式失败。
  - 数字型 `size` 值必须为正数。
  - 命名尺寸值必须来自 package 自有的 `IconSize` 集合。
  - 当省略 `label` 时，渲染结果必须从辅助技术中隐藏。

## 实体：图标预览示例

- **目的**：表示 Storybook 与 website 指导中一个可评审的图标示例。
- **字段**：
  - `exampleId`：稳定示例标识符，用于文档和测试引用
  - `scenario`：该示例展示的使用场景
  - `iconNames`：示例中出现的一个或多个目录项
  - `statesCovered`：示例覆盖的无标签、有标签、尺寸、主题和失败场景
  - `notes`：与该示例相关的贡献者指导
- **校验规则**：
  - 每个基础分类都必须至少在一个预览示例中出现。
  - 失败场景示例必须展示不支持图标名时的显式行为。

## 状态迁移

- `active -> deprecated`：允许，但必须提供替代项或迁移说明。
- `deprecated -> removed`：允许，但前提是已存在迁移指导，并且移除行为在发布前已被明确文档化。
- `active -> removed`：对于基础目录不推荐，且必须在发布前明确记录，因为这会破坏稳定的消费者契约。
