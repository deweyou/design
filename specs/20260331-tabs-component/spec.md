**功能分支**：`20260331-tabs-component`  
**创建时间**：2026-03-31  
**状态**：草稿  
**输入**：用户描述："基于arkUI 的tabs组件，实现当前组件库的 Tabs 组件，结合当前仓库的其他组件的设计风格实现。需要对齐 arkUI Tabs 组件的基础能力，同时支持TabItem 承接 Menus 下拉框（竖形就右边出现）。然后支持线条的active样式和hover bg的active样式，线条的话支持切换动画，就是线条在pre active 滑动到 next active 的动画，竖形的话线条可以在左侧，borderleft 的位置，然后上下滑动动画。然后默认会支持每个tab 的 content内容的，支持关闭，就仅tabs。然后支持超长滑动case，超长支持滑动，且左右有渐变样式，触边就不渐变；也可以支持超长收齐，最后一个是个下拉 menu。"

## 用户场景与测试（必填）

### 用户故事 1 - 基础标签页切换（优先级：P1）

开发者在页面中使用 Tabs 组件切换不同内容区域。用户点击标签项，对应内容面板出现，激活状态通过下划线指示器（横排）或左边框指示器（竖排）标识，指示器在切换时平滑滑动。未激活标签在 hover 时有背景色反馈。

**为什么是这个优先级**：这是 Tabs 组件的核心价值，所有其他能力都建立在基础切换功能之上。

**独立测试**：在 website 中渲染横排和竖排 Tabs，分别点击各 tab，验证内容切换正确、指示器动画流畅、hover 状态可见。

**验收场景**：

1. **假如** 渲染了一个横排 Tabs（默认），**当** 用户点击第 2 个 tab，**那么** 第 2 个 tab 内容面板变为可见，下划线指示器从第 1 个 tab 平滑滑动到第 2 个 tab 位置。
2. **假如** 渲染了一个竖排 Tabs（`orientation="vertical"`），**当** 用户点击第 3 个 tab，**那么** 左侧竖线指示器从第 2 个 tab 平滑上下滑动到第 3 个 tab 位置。
3. **假如** 任意 tab 处于非激活状态，**当** 用户将鼠标移动到该 tab 上，**那么** 该 tab 显示半透明背景色反馈（hover bg 样式）。
4. **假如** Tabs 处于 `indicator` 样式变体，**当** 用户快速连续切换多个 tab，**那么** 指示器始终跟随最后点击的 tab 平滑运动，不出现闪烁或跳跃。

---

### 用户故事 2 - 仅标签栏模式（无内容面板）（优先级：P1）

开发者需要一个纯标签导航栏，不需要 Tabs 组件管理内容面板（例如路由驱动的页面切换场景）。

**为什么是这个优先级**：导航场景中 Tabs 通常仅作为选择器存在，内容由路由管理，这是常见的真实用例。

**独立测试**：在 website 中使用 `hideContent` 或等效 prop 渲染一个纯标签栏，确认无内容面板渲染，激活状态与指示器仍正常工作。

**验收场景**：

1. **假如** Tabs 配置为仅标签栏模式（关闭内容面板），**当** 页面渲染，**那么** DOM 中不存在内容面板区域，仅标签列表可见。
2. **假如** 仅标签栏模式下，**当** 用户切换 tab，**那么** `onValueChange` 回调被触发，指示器动画正常，外部可响应切换事件。

---

### 用户故事 3 - Tab 下拉菜单（TabItem 承接 Menu）（优先级：P2）

某个 tab 项自身携带一个子菜单，点击后展开下拉选项而不是直接切换内容面板。横排模式下拉菜单向下展开，竖排模式下拉菜单向右展开。

**为什么是这个优先级**：复合场景，提升 Tabs 的表达能力，允许在标签栏中内嵌分组导航，但不影响基础切换功能的交付。

**独立测试**：在 website 中渲染一个含 Menu 类型 tab 的 Tabs，点击该 tab 验证下拉菜单弹出方向正确，从菜单中选择一项后激活对应内容。

**验收场景**：

1. **假如** 一个 tab 项配置了子菜单选项，**当** 用户在横排模式下点击该 tab，**那么** 下拉菜单向下弹出，列出所有子选项。
2. **假如** 一个 tab 项配置了子菜单选项，**当** 用户在竖排模式下点击该 tab，**那么** 下拉菜单向右弹出。
3. **假如** 下拉菜单已展开，**当** 用户点击某个子选项，**那么** 菜单收起，对应内容面板被激活，激活指示器更新到该 tab。
4. **假如** 下拉菜单已展开，**当** 用户点击菜单外部或按 `Escape`，**那么** 菜单收起，无激活状态变化。

---

### 用户故事 4 - 标签栏超长滚动模式（优先级：P2）

Tab 数量超过标签栏可显示宽度时，标签栏支持横向（横排）或纵向（竖排）滚动，并在可滚动侧边呈现渐变遮罩；当滚动到边缘时，对应侧渐变消失。

**为什么是这个优先级**：当 tab 数量不确定时，必须有降级方案防止布局溢出，是实际项目中的高频需求。

**独立测试**：在 website 中渲染超过 10 个 tab，验证标签栏出现渐变遮罩，滑动到两端时对应侧渐变消失。

**验收场景**：

1. **假如** tab 总宽度超出容器，**当** 页面渲染且默认激活第一个 tab，**那么** 标签栏右侧显示渐变遮罩，左侧不显示渐变（已到边缘）。
2. **假如** 处于超长滚动状态，**当** 用户向右滚动到中间位置，**那么** 左右两侧均显示渐变遮罩。
3. **假如** 处于超长滚动状态，**当** 用户滚动到最右侧边缘，**那么** 右侧渐变消失，左侧渐变保持。
4. **假如** 激活的 tab 当前不在可视区域，**当** 切换到该 tab，**那么** 标签栏自动滚动使该 tab 可见。

---

### 用户故事 5 - 标签栏超长收齐模式（优先级：P3）

Tab 数量超长时，提供另一种策略：仅展示能放下的 tab，其余 tab 收入末尾的"更多"下拉菜单中。

**为什么是这个优先级**：在固定布局（不可滚动）场景下提供优雅降级，使标签栏始终紧凑，但实现复杂度高，优先级低于滚动方案。

**独立测试**：在 website 中以收齐模式渲染超长 Tabs，验证末尾出现"更多"按钮，点击后下拉菜单列出溢出的 tab，选中后激活状态正确。

**验收场景**：

1. **假如** Tabs 配置为超长收齐模式（`overflowMode="collapse"`），且 tab 数量超出容器宽度，**当** 页面渲染，**那么** 标签栏末尾出现"更多"下拉按钮，其余放不下的 tab 不直接显示。
2. **假如** 收齐模式下"更多"按钮可见，**当** 用户点击"更多"，**那么** 下拉菜单列出所有溢出的 tab 标题。
3. **假如** 当前激活的 tab 位于溢出列表中，**当** 页面渲染，**那么** "更多"按钮呈激活视觉状态，提示用户当前位置在折叠区域。
4. **假如** 从"更多"菜单中选择一个 tab，**当** 用户点击，**那么** 该 tab 被激活，内容切换，菜单收起。

---

### 边界情况

- 当只有一个 tab 时，Tabs 仍正常渲染，指示器固定在唯一 tab 下，无切换动画。
- 当某个 tab 被标记为 `disabled` 时，点击无效，键盘导航跳过该 tab，视觉上呈半透明禁用状态（`opacity: 0.56`）。
- 仅键盘用户通过方向键在 tab 间导航，通过 `Enter`/`Space` 激活（手动激活模式）或 focus 即激活（自动激活模式），`Tab` 键可进入内容面板。
- 超长滚动模式下，激活被收进视口外的 tab 时，标签栏自动滚动使其可见。
- 在 `prefers-reduced-motion` 媒体查询生效时，指示器切换动画禁用（直接跳切），其余交互反馈保持。
- Tabs 组件应在浅色/深色主题 token 下均能正确显示（复用设计系统 token，不硬编码颜色）。
- 竖排模式下，Tab 下拉菜单弹出方向为右侧，避免与左侧指示器视觉冲突。

## 需求（必填）

### 功能需求

**核心行为（对齐 Ark UI Tabs 基础能力）**

- **FR-001**：组件必须支持受控模式（`value` + `onValueChange`）和非受控模式（`defaultValue`），行为与 Ark UI `Tabs.Root` 对齐。
- **FR-002**：组件必须支持 `orientation` 属性，取值 `horizontal`（默认）和 `vertical`，控制标签栏排列方向及键盘导航轴向。
- **FR-003**：组件必须支持 `activationMode` 属性，取值 `automatic`（focus 即激活，默认）和 `manual`（需按 Enter/Space）。
- **FR-004**：组件必须支持 `loopFocus` 属性（默认 `true`），控制键盘导航是否在首尾循环。
- **FR-005**：组件必须支持单个 tab 的 `disabled` 状态，禁用 tab 不可点击、不可键盘激活，视觉呈禁用态。
- **FR-006**：组件必须支持 `lazyMount` 和 `unmountOnExit` 属性，控制非激活内容面板的挂载/卸载行为。

**视觉样式变体**

- **FR-007**：组件必须支持两种激活指示器样式变体，通过 `variant` 属性控制：
  - `line`（默认）：横排时为标签底部下划线，竖排时为标签左侧竖线（border-left）。
  - `bg`：激活 tab 背景填充（hover bg 加强版），无滑动指示器线条。
- **FR-008**：`line` 变体的指示器必须支持平滑切换动画：横排时水平滑动，竖排时垂直滑动；动画时长 160ms，缓动曲线与仓库动效规范一致。
- **FR-009**：所有 tab 在 hover 时必须显示半透明背景色反馈（使用 `color-mix` 透明混色方案，与仓库 hover 规范一致）。
- **FR-010**：组件必须支持 `size` 属性（至少 `small`、`medium`、`large` 三档），控制 tab 的字号、内边距等尺寸。

**Tab 下拉菜单**

- **FR-011**：单个 tab 项必须支持配置子菜单（`menuItems` prop 或等效 API），将该 tab 渲染为带下拉菜单的触发器。
- **FR-012**：横排模式下，tab 子菜单向下弹出；竖排模式下，tab 子菜单向右弹出。
- **FR-013**：从子菜单中选择一项后，对应内容面板被激活，菜单关闭，激活指示器更新。
- **FR-014**：带子菜单的 tab 本身视觉上应有展开方向指示（如箭头图标），并正确响应无障碍属性（`aria-haspopup`、`aria-expanded`）。

**内容面板控制**

- **FR-015**：组件默认渲染各 tab 对应的内容面板（`TabContent`），支持通过 `hideContent` prop（或等效）关闭内容面板区域，仅保留标签栏。
- **FR-016**：关闭内容面板时，`onValueChange` 回调仍正常触发，激活指示器仍正常工作。

**超长溢出处理**

- **FR-017**：组件必须支持 `overflowMode` 属性，取值：
  - `scroll`（默认）：tab 超长时，标签栏可横向（横排）或纵向（竖排）滚动。
  - `collapse`：tab 超长时，溢出 tab 收入末尾"更多"下拉菜单。
- **FR-018**：`scroll` 模式下，标签栏在可滚动的一侧显示渐变遮罩（宽度约 40px），到达边缘时对应侧渐变消失。
- **FR-019**：`scroll` 模式下，切换到视口外的 tab 时，标签栏自动滚动使目标 tab 可见。
- **FR-020**：`collapse` 模式下，末尾"更多"按钮必须打开一个下拉菜单，列出所有当前溢出的 tab；当前激活 tab 若在溢出列表中，"更多"按钮呈激活视觉状态。
- **FR-020a**：`collapse` + `line` 变体组合下，当激活 tab 切换至溢出区域时，滑动指示器须向右滑向"更多"按钮方向并收缩至零宽（不得向左跳至原点）；从溢出区域返回可见 tab 时，指示器须从"更多"按钮侧向目标 tab 滑入。实现上须排除 `display:none` 的溢出 tab（其 `offsetLeft = 0` 会导致动画方向错误）。

**公开 API 面**

- **FR-021**：组件必须定义清晰的公开 API，包括：`Tabs`（根容器）、`TabList`（标签列表）、`TabTrigger`（单个标签触发器）、`TabContent`（内容面板）、`TabIndicator`（指示器，可选组合使用）；API 与 Ark UI 原语解耦，不直接透传 Ark UI 内部 props。
- **FR-022**：受控/非受控行为对消费者透明，通过 `value` 与 `defaultValue` 二选一使用。

**无障碍**

- **FR-023**：`TabList` 必须带 `role="tablist"` 及 `aria-orientation` 属性；每个 `TabTrigger` 带 `role="tab"`、`aria-selected`、`aria-controls`；每个 `TabContent` 带 `role="tabpanel"`、`aria-labelledby`；焦点管理符合 ARIA APG Tabs Pattern。
- **FR-024**：键盘导航：方向键在 tab 间移动焦点，`Home`/`End` 跳至首尾，`Tab` 键将焦点移入激活内容面板。

**设计 Token 与样式**

- **FR-025**：组件样式完全通过 CSS Modules（Less）+ 现有设计 token 实现，不引入新的颜色 token；尺寸档位如需新增 token，须在 plan 中列出。
- **FR-026**：`prefers-reduced-motion` 生效时，指示器切换动画禁用（`transition: none`），其余交互不受影响。

**预览覆盖**

- **FR-027**：必须在 `website` 和/或 `storybook` 中提供以下场景预览：横排基础、竖排基础、`line` 变体、`bg` 变体、含 Menu 的 tab、超长滚动模式、超长收齐模式、仅标签栏模式、禁用 tab。

### 无障碍与 UI 契约

- **用户角色 / 参与方**：仅键盘用户、屏幕阅读器用户（NVDA/VoiceOver）、消费该 package 的应用开发者、验证 token 的设计师。
- **交互模型**：点击激活、方向键导航、`Home`/`End` 跳转、`Tab` 进入内容区、`Escape` 关闭子菜单、鼠标滚轮横向滚动标签栏（`scroll` 模式）。
- **需要覆盖的状态**：default（未激活）、active（激活）、hover、focus-visible、disabled、menu-open（tab 带子菜单展开时）、overflow-scroll（标签栏可滚动渐变态）、overflow-collapse（更多菜单）。
- **主题 / token 影响**：复用现有语义色 token（`--ui-color-brand-bg`、`--ui-color-text`、`--ui-color-border`、`--ui-color-focus-ring`、`--ui-color-surface`）；指示器颜色通过 `color` prop（`neutral` / `primary`）控制；尺寸 token 视 plan 阶段确认是否新增。

### 关键实体

- **Tab**：单个标签项，属性包括 `value`（唯一标识）、`label`（显示文本）、`disabled`（是否禁用）、`menuItems`（可选子菜单列表）。
- **TabList**：标签容器，负责排列 Tab，管理溢出策略（`scroll` / `collapse`）和渐变遮罩状态。
- **TabContent**：与单个 Tab 关联的内容面板，通过 `value` 与 Tab 对应；可整体隐藏。
- **TabIndicator**：激活指示器，在 tab 间平滑移动；横排为底部线，竖排为左侧线；`line` 变体专属。
- **OverflowMenu**：超长收齐模式下的"更多"下拉菜单，动态收容当前溢出的 Tab 列表。

## 成功标准（必填）

### 可度量结果

- **SC-001**：组件覆盖 FR-001～FR-026 所有功能需求，在 website 中可交互演示所有主要场景。
- **SC-002**：指示器切换动画在现代浏览器中（Chrome / Safari / Firefox 最新两版）流畅运行，无明显掉帧（目视 60fps 无卡顿）。
- **SC-003**：所有核心交互场景（基础切换、键盘导航、下拉菜单、溢出模式）有自动化测试覆盖，测试套件通过率 100%。
- **SC-004**：`scroll` 模式下渐变遮罩能准确反映当前滚动位置（到边缘即消失），用户在视觉上可判断是否还有更多 tab。
- **SC-005**：`collapse` 模式下，"更多"菜单中的 tab 可被正常激活，激活状态与直接点击 tab 行为一致。
- **SC-006**：组件在仅键盘操作场景下，所有 tab 均可通过方向键访问并激活，无需鼠标介入。
- **SC-007**：在 `prefers-reduced-motion` 环境下，组件功能完整，动画禁用后无布局错位或功能回归。

## 假设与约定

- Tab 子菜单（`menuItems`）复用或参考现有 Popover/Menu 组件的弹出定位机制，不自行实现定位逻辑。
- `collapse` 模式下的溢出检测基于容器宽度变化（ResizeObserver），不依赖固定断点。
- 渐变遮罩使用 CSS `mask-image` 线性渐变实现，兼容性目标与仓库现有 CSS 特性使用范围一致。
- Tab 的 `color` prop 默认为 `neutral`，`primary` 时指示器和激活文字使用品牌色（`--ui-color-brand-bg`）。
- 不在本次范围内：tab 拖拽排序、可关闭（closable）tab 标签、tab 懒加载内容的骨架屏。
