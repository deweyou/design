# 归档：npm 发包工作流

**分支**：`20260408-npm-publish-workflow`  
**完成时间**：2026-04-08  
**类型**：build

---

## 交付摘要

建立了双轨发包工作流：非 `main` 分支发布带 `-beta.N` 后缀的预发布包到 `beta` dist-tag，`main` 分支发布纯语义化版本到 `latest` dist-tag，两者均有分支校验门控防止在错误分支发布。引入约定式提交（Conventional Commits）驱动的版本号自动推断（`fix`→patch / `feat`→minor / `BREAKING CHANGE`→major）和 `CHANGELOG.md` 自动生成，monorepo 中只升级有实际变更的包。发布前强制全量构建并校验 npm 鉴权状态。

---

## 关键决策

| 决策                  | 选择           | 理由                                   | 备选方案           |
| --------------------- | -------------- | -------------------------------------- | ------------------ |
| 预发布标识格式        | `-beta.N`      | 语义清晰，与语义化版本规范兼容         | `-rc.N`/`-alpha.N` |
| 版本推断来源          | 约定式提交类型 | 项目已强制 commit-msg 格式，可直接依赖 | 手动指定版本号     |
| `packages/infra` 发包 | 不发布到 npm   | 纯构建时工具，不是面向消费者的运行时包 | 一同发布           |

## 可复用模式

- [双轨发布模型]：非 `main` 分支 → `beta` 预发布，`main` 分支 → `stable` 正式发布，分支校验作为门控，是本仓库的标准发版约定。
