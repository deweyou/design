# 归档：按钮 Hover 反馈整理

**分支**：`20260323-button-hover-motion`  
**完成时间**：2026-03-23  
**类型**：refactor

---

## 交付摘要

整理了 `link` 和 `outlined` 两个 Button variant 的 hover 反馈：`link` 改为默认内建从左到右的下划线显现动画（260ms ease，clip-path 展开），`outlined` 取消环绕描边动画，改为真实 border 的颜色过渡（默认低色度 border，hover 时平滑过渡到与文字一致的颜色），同时移除 `animated` 公开属性，hover 行为直接内建到 variant 中不再提供开关。

---

## 关键决策

| 决策                   | 选择                 | 理由                                                  | 备选方案                   |
| ---------------------- | -------------------- | ----------------------------------------------------- | -------------------------- |
| hover 反馈是否公开开关 | 不提供开关，直接内建 | 减少 API 复杂度，保持一致性                           | 保留 `animated` prop       |
| `outlined` hover 实现  | 真实 border 颜色过渡 | 避免额外装饰层与真实 border 产生间距/圆角错位         | 环绕描边动画层             |
| `link` 下划线实现      | `clip-path` 展开动画 | 可精确控制显现方向，避免与原生 `text-decoration` 叠加 | CSS `text-decoration` 动画 |

---

## 踩坑记录

- **问题**：`link` icon + text 组合时，下划线只贴在文字片段下方而非整条内容行。  
  **解决方案**：下划线元素覆盖整条内容行宽度，不仅限于文字节点范围。

---

## 可复用模式

- [内建 hover 反馈，不暴露开关]：hover 动效作为 variant 默认行为内建，后续组件可沿用此模式避免多余的 `animated` 类 props 污染 API。
