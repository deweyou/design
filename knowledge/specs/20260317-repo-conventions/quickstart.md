# 快速开始：仓库规范统一

## 目标

落地仓库治理规则，使用户要求的规范在编码时可见，并在可行范围内通过自动化执行。

## 推荐实现顺序

1. 更新根级治理文件：
   - 完善根级 `AGENTS.md`，文档化整个仓库的编写与结构规则。
   - 对齐 `components`、`hooks`、`utils` 的 package 级 `AGENTS.md`，确保 package 本地要求与仓库策略一致。

2. 为机械性规则增加自动化：
   - 扩展仓库的 lint 或 check 配置，覆盖那些可以可靠验证的规则。
   - 优先限制受治理 React 代码中的 `createElement` 使用，以及可安全表达的函数风格约束。
   - 仅对明确受治理的 package 区域增加路径校验，避免对历史代码产生误报。

3. 对齐示例与预览面：
   - 移除 `apps/website` 或其他可见示例中与规范冲突的写法。
   - 确保 package 变更后，组件预览要求仍然完整。

4. 渐进迁移源码布局：
   - 优先对新增或有实质性修改的 utilities、hooks 和 components 应用新目录结构。
   - 对仍需后续治理的历史路径做记录，而不是在首个变更中强制大规模搬迁。

## 验证

运行：

```sh
vp check
vp test
```

然后人工确认：

- 根级和 package 级 `AGENTS.md` 指导保持一致。
- demo apps 不再把被禁止模式当作推荐做法展示。
- 受治理 packages 为共置测试和单元目录提供了清晰路径。

## 预期结果

贡献者应当能够仅根据仓库指导推断出正确的编写风格与目录结构，同时由自动化检查在评审前拦住机械性违规。
