# 归档：仓库规范统一

**分支**：`20260317-repo-conventions`  
**完成时间**：2026-03-17  
**类型**：chore

---

## 交付摘要

确立了仓库级代码规范：函数默认使用箭头函数，React 组件统一使用 TSX 文件，受治理区域的文件和目录命名使用小写连字符（kebab-case）。明确 `packages/react`、`packages/react-hooks`、`packages/infra` 下每个源码单元位于 `src/<unit-name>/` 目录，并将 `index` 入口与 `index.test` 单测共置于同目录，废除并行的顶层测试目录。同时建立了 commit message 格式约定，通过 `.vite-hooks/commit-msg` 强制校验。

---

## 关键决策

| 决策     | 选择               | 理由                                                   | 备选方案                  |
| -------- | ------------------ | ------------------------------------------------------ | ------------------------- |
| 单测位置 | 与源码共置于同目录 | 降低测试文件游离概率，便于修改时一并评审               | 并行顶层 `test/` 目录     |
| 函数风格 | 默认箭头函数       | 统一风格、减少评审歧义；框架边界等场景允许例外并需说明 | 混用函数声明              |
| 组件格式 | 强制 TSX           | 工具链（类型检查、JSX 转换）与规范更一致               | `.jsx` 或 `createElement` |
