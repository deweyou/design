# 归档：Storybook E2E 测试覆盖

**分支**：`20260408-storybook-e2e`  
**完成时间**：2026-04-08  
**类型**：test

---

## 交付摘要

在 `apps/storybook` 中引入 `@storybook/test-runner`（基于 Playwright），为所有现有组件 story 建立 E2E 测试套件。交互型组件（Button、Popover、Menu、Tabs）覆盖核心交互路径、键盘导航和关键 ARIA 属性验证；展示类组件（Typography、Color、Icon）提供渲染正确性 smoke test。测试支持 CI 无头运行，通过 `vp run storybook#test` 集成进现有工作流，测试失败时 CI 退出码非零。

---

## 关键决策

| 决策         | 选择                                         | 理由                                                         | 备选方案                  |
| ------------ | -------------------------------------------- | ------------------------------------------------------------ | ------------------------- |
| E2E 框架     | `@storybook/test-runner` + Playwright        | 与 Storybook 原生集成，无需额外测试服务器                    | Cypress + Storybook       |
| 测试范围     | 行为正确性 + ARIA 属性，不含视觉回归         | 行为回归是最高频问题，视觉回归成本更高且需要额外维护截图基准 | 同时引入视觉回归          |
| 测试代码位置 | 仅在 `apps/storybook` 内，不侵入 `packages/` | 保持组件包纯净，测试基础设施与组件实现解耦                   | 在各组件包中添加 E2E 测试 |
