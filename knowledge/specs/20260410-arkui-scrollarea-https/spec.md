# 功能规格：ScrollArea 滚动区域组件

**功能分支**：`20260410-arkui-scrollarea-https`  
**创建时间**：2026-04-10  
**状态**：已完成  
**输入**：用户描述："基于 arkUI 的 scrollArea 组件（https://ark-ui.com/docs/components/scroll-area），实现我的设计风格的 srcollArea 组件。"  
**语言要求**：本文件及同级 `/specs/` 文档正文必须使用简体中文；代码标识符、命令、
文件路径、协议字段和第三方 API 名称可保留原文。

## 假设

- 滚动条默认仅在垂直方向显示；水平滚动作为可选变体通过 props 开启。
- 滚动条采用**叠层（overlay）模式**：绝对定位浮于内容上方，不占用容器的布局宽度或高度。
- 滚动条在内容无溢出时自动隐藏，有溢出时显示（通过 `[data-overflow-y]` / `[data-overflow-x]` 控制）。
- 默认不限制滚动区域高度——消费者通过容器约束高度（组件本身不内置固定高度）。
- 滚动条轨道和滑块遵循本项目既有交互数值规范（140ms 过渡、disabled opacity: 0.56 等）。
- ScrollArea 作为展示型容器组件，不需要 `variant` / `color` / `shape` / `size` 四维变体模型——只保留 `color` 维度控制滑块颜色语义。
- 组件支持 `autoHide` 选项：鼠标离开区域后延迟 1500ms 自动隐藏滚动条，通过纯 CSS `transition-delay` 实现，不引入 JS 计时器。

## 用户场景与测试（必填）

### 用户故事 1 - 垂直滚动区域（优先级：P1）

开发者将一段超出容器高度的内容放入 ScrollArea 后，用户可通过鼠标滚轮或拖拽滑块上下滚动内容，滚动条以符合项目设计风格的样式呈现，且在内容不溢出时自动隐藏。

**为什么是这个优先级**：这是 ScrollArea 的核心场景，覆盖最主要的使用场景。

**独立测试**：可渲染一个高度受限的容器内包含超长文本，验证滚动条出现、可拖拽且样式符合设计规范，即可完整交付 MVP。

**验收场景**：

1. **假如** 内容高度超过容器高度，**当** 页面加载，**那么** 右侧出现垂直滚动条，颜色、圆角、宽度符合设计风格。
2. **假如** 内容高度未超过容器高度，**当** 页面加载，**那么** 滚动条不可见。
3. **假如** 垂直滚动条可见，**当** 用户拖拽滑块，**那么** 内容随之滚动。
4. **假如** 用户悬停在滚动条上，**当** 鼠标进入，**那么** 滑块呈现 hover 加深样式（transition 140ms ease）。

---

### 用户故事 2 - 水平滚动区域（优先级：P2）

开发者将超出容器宽度的内容放入 ScrollArea 并开启水平模式，用户可横向滚动内容，水平滚动条以底部呈现，样式一致。

**为什么是这个优先级**：水平滚动在表格、横排卡片等场景常见，是重要的补充用例，但相对垂直滚动频率低。

**独立测试**：渲染宽度受限容器内包含超宽内容，开启水平滚动选项，验证底部出现滚动条且可拖拽。

**验收场景**：

1. **假如** 内容宽度超过容器宽度且水平滚动已启用，**当** 页面加载，**那么** 底部出现水平滚动条。
2. **假如** 水平与垂直滚动同时启用且均有溢出，**当** 页面加载，**那么** 右侧垂直和底部水平滚动条同时显示，右下角呈现 Corner 填充块。

---

### 用户故事 3 - 程序式滚动控制（优先级：P3）

开发者需要通过代码将滚动区域滚到顶部或底部（如"回到顶部"按钮），可通过暴露的 `ref` 或受控 API 实现。

**为什么是这个优先级**：程序式控制在页面级滚动交互中有价值，但属于进阶用例。

**独立测试**：在 `website` 预览中添加"滚动到顶部"和"滚动到底部"按钮，验证点击后内容正确定位。

**验收场景**：

1. **假如** 内容已滚动到中间位置，**当** 开发者调用滚动到顶部操作，**那么** 内容平滑回到顶部。

---

### 边界情况

- 当内容动态变化（如异步追加内容）时，滚动条会否自动重新计算并显示？
- 嵌套 ScrollArea（ScrollArea 内部包含另一个 ScrollArea）时，滚动事件如何传递？
- 仅使用键盘的用户如何聚焦和操作滚动区域内的内容？（滚动本身由浏览器原生键盘事件处理，无需额外实现）
- 内容量为零或极少时，滚动条正确隐藏。
- 在极窄或极矮的容器中，滑块最小长度有保证（避免滑块消失）。

## 需求（必填）

### 功能需求

- **FR-001**：系统必须提供 `ScrollArea` 复合组件，作为 `packages/react` 中的独立源码单元（`src/scroll-area/`）发布。
- **FR-002**：系统必须基于 `@ark-ui/react` 的 `ScrollArea` 原语实现行为层，样式层使用 CSS Modules（Less）+ 设计 token。
- **FR-003**：系统必须支持垂直滚动（默认）和水平滚动（可选），通过 props 控制启用哪个方向的滚动条。
- **FR-004**：滚动条必须采用叠层（overlay）模式，绝对定位浮于内容之上，不占用任何布局空间；内容区域的可用宽度/高度不因滚动条存在而缩窄。
- **FR-004b**：滚动条在内容无溢出时自动隐藏，有溢出时显示，显隐过渡动效遵循 140ms ease。
- **FR-005**：系统必须支持 `color` 维度（`primary` / `neutral` 两档），控制滚动条滑块颜色：`primary` 为品牌绿（默认），`neutral` 使用 `currentColor` 继承父容器 `color` 属性——在深色背景容器中自动呈浅色，无需消费方手动切换。
- **FR-010**：系统必须支持 `autoHide` prop（`boolean`，默认 `false`）：启用后，鼠标离开滚动区域 1500ms 后滚动条渐隐（纯 CSS `transition-delay`，无 JS 计时器）；鼠标在区域内时恢复 0.4 默认透明度，hover 到轨道时恢复不透明。
- **FR-006**：系统必须为 `packages/react` 定义公开 API，包括 `ScrollArea`（复合对象或单一组件）、Props 类型，以及是否暴露 `ref` 和程序式控制 API。
- **FR-007**：滚动内容区域必须可被键盘和辅助技术访问——内部内容的焦点顺序不受滚动容器干扰，滚动区域本身不拦截 Tab 键。
- **FR-008**：所有滑块、轨道视觉样式必须通过 `--ui-*` token 表达，不得硬编码颜色或圆角数值；hover、active 状态使用 `color-mix` 混色规范（6-12%）；滑块禁用态使用 `opacity: 0.56`；过渡值为 `140ms ease`。
- **FR-009**：系统必须在 `website` 中提供以下预览：垂直滚动默认态、水平滚动态、双向同时滚动、color 两档（primary / neutral，含深色背景对比）、autoHide 演示，以及内容不溢出时滚动条隐藏的状态。

### 无障碍与 UI 契约（UI 工作必填）

- **用户角色 / 参与方**：
  - 鼠标用户（拖拽滑块、滚轮）
  - 触控用户（触摸滑动）
  - 键盘用户（Tab 穿透内容，方向键在内部元素上操作）
  - 屏幕阅读器用户（不感知滚动条存在，只消费内部内容）
  - 消费该 package 的应用开发者（通过 props 配置）

- **交互模型**：
  - 鼠标拖拽滑块滚动
  - 点击轨道分页滚动
  - 鼠标滚轮/触控板自然滚动
  - 滚动条对屏幕阅读器隐藏（`aria-hidden`），不干扰内容无障碍树

- **需要覆盖的状态**：
  - 默认（内容溢出，滚动条可见）
  - 内容未溢出（滚动条隐藏）
  - 滚动条 hover
  - 滚动条 active（正在拖拽）
  - 双向滚动（水平 + 垂直同时显示，Corner 填充）

- **主题 / token 影响**：
  - `primary`：复用 `--ui-color-brand-bg`（浅色 emerald-700，深色 emerald-700，品牌色）
  - `neutral`：使用 `currentColor`（继承父容器 `color` 属性，深色容器自动呈浅色，无需主题感知逻辑）
  - 轨道背景透明（overlay 模式下轨道不可见）

### 关键实体

- **ScrollArea（滚动区域）**：包含一个固定尺寸（高度/宽度由外部约束）的视口区域，内部内容可超出视口并通过滚动条访问。关键属性：方向（垂直/水平/双向）、尺寸档位。
- **Scrollbar（滚动条）**：由轨道（track）和滑块（thumb）组成的交互控件。关键属性：方向（`orientation`）、可见性（由溢出状态决定）。
- **Thumb（滑块）**：反映当前滚动位置的可拖拽控件，长度与可见内容占总内容比例成正比。

## 成功标准（必填）

### 可度量结果

- **SC-001**：开发者可以用不超过 3 行 JSX 将任意内容包裹进 ScrollArea 并获得完整的样式化滚动体验，无需手写 CSS。
- **SC-002**：在内容有溢出和无溢出两种状态下，滚动条的显隐切换过渡流畅（140ms），无闪烁或布局抖动。
- **SC-003**：所有可见交互状态（默认、hover、active、隐藏）及两档 color（primary / neutral）在 `website` 预览中均可独立评审。
- **SC-004**：组件在键盘导航时不拦截 Tab 焦点，屏幕阅读器不会额外宣读滚动条元素。
- **SC-005**：主要组件状态（溢出/不溢出、垂直/水平/双向、color 两档）被自动化测试覆盖，并可在 `website` 中进行评审。
