# 归档：包构建与发布产物治理

**分支**：`20260326-optimize-package-outputs`  
**完成时间**：2026-03-26  
**类型**：build

---

## 交付摘要

完成了三项包产物治理工作：（1）为 `@deweyou-design/react` 中每个公开组件提供独立消费入口，通过 `preserveModules` + CSS split 实现按需导入且无需额外导入样式；（2）确立 Vite+ 统一构建约定为默认方案，仅在有明确必要性时保留包级专用构建配置，并将规则文档化；（3）要求所有拟发布包在产物中消除 `workspace:*` 和 `catalog:` 占位符，`react`/`react-dom` 声明为 `peerDependencies` 由宿主安装，内部包依赖解析为具体版本号。

---

## 关键决策

| 决策                     | 选择                             | 理由                               | 备选方案           |
| ------------------------ | -------------------------------- | ---------------------------------- | ------------------ |
| 组件样式导入             | 单组件入口自带样式，无需额外导入 | 降低消费心智负担，避免遗漏样式导入 | 消费者手动导入样式 |
| 包级构建配置             | 默认不保留，例外需说明           | 减少重复配置，降低新包接入成本     | 每包独立配置       |
| `react`/`react-dom` 依赖 | `peerDependencies`               | 运行时由宿主提供，避免版本冲突     | `dependencies`     |

## 可复用模式

- [发布依赖契约]：区分 `peerDependencies`（宿主安装）、`dependencies`（运行时自带）、`devDependencies`（仅构建期）三类，是本仓库所有可发布包的标准分类约定。
- [产物版本占位符校验]：发布前检查 `dist/package.json` 不含 `workspace:*`/`catalog:` 是发版流程的必要门控。
