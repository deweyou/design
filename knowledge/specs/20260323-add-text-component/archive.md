# 归档：Text 排版组件

**分支**：`20260323-add-text-component`  
**完成时间**：2026-03-23  
**类型**：feat

---

## 交付摘要

新增 `Text` 组件，支持 8 种 `variant`（`plain`/`body`/`caption`/`h1`–`h5`），通过 `italic`、`bold`、`underline`、`strikethrough` 四个可叠加布尔字段表达文本装饰，`color` 和 `background` 基于 26 个颜色家族的共享色卡 token 暴露预定义 union，不接受任意自定义色值，主题切换时自动映射深浅色阶。`lineClamp` 限制最大可见行数，所有常见节点属性透传到最终渲染节点。

---

## 关键决策

| 决策                      | 选择                                                                                    | 理由                                                   | 备选方案         |
| ------------------------- | --------------------------------------------------------------------------------------- | ------------------------------------------------------ | ---------------- |
| `color`/`background` 粒度 | 暴露 26 个颜色家族名 union，不暴露色阶                                                  | 消费方无需关心 `50–950` 具体色阶，由系统按主题自动决定 | 直接暴露色阶编号 |
| 不引入 `as` 多态          | 各 variant 固定渲染节点（`plain`→`span`、`body`/`caption`→`div`、`h1–h5`→对应标题节点） | 降低 API 复杂度，保持节点语义与视觉层级一致            | 通用 `as` prop   |
| `caption` 弱化实现        | 透明度或等价弱化方式，保持可读                                                          | 简单且可在现有 token 体系内实现                        | 独立颜色 token   |

## 可复用模式

- [颜色家族 union]：`color`/`background` 仅暴露家族名、不暴露色阶的模式可被后续任何需要语义颜色的组件复用。
- [节点属性透传]：所有 HTML 节点属性（`aria-*`、`data-*`、`className` 等）直接透传到最终渲染节点，是本仓库组件的标准透传约定。
