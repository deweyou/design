# 归档：引入 Ark UI 作为组件基础层

**分支**：`20260327-ark-ui-integration`  
**完成时间**：2026-03-27  
**类型**：refactor

---

## 交付摘要

将 `@ark-ui/react` 确立为本组件库中复杂交互组件的开发基础层，避免重复实现已有成熟解法的行为逻辑（无障碍、键盘导航、焦点管理等）。将现有 `Popover` 组件基于 Ark UI 对应原语重新实现，保留现有公开 API 和设计风格，消费方无需修改任何调用代码。在 `CLAUDE.md` 和 `knowledge/constitution.md` 中新增明确条款：交互型新组件优先基于 Ark UI 原语开发，仅在 Ark UI 无对应覆盖时才考虑自行实现。

---

## 关键决策

| 决策                     | 选择                                     | 理由                                   | 备选方案                   |
| ------------------------ | ---------------------------------------- | -------------------------------------- | -------------------------- |
| `Button`/`Text` 是否迁移 | 不迁移                                   | 两者行为简单，不需要 headless 层介入   | 全量迁移                   |
| Ark UI 使用范围          | 有对应原语的交互型组件                   | 行为逻辑复杂度是使用 Ark UI 的合理门槛 | 所有组件强制使用           |
| API 契约                 | 保持与重构前完全兼容，无 breaking change | 消费方无感知迁移是核心前提             | 允许小范围 breaking change |
