# 功能规格：补齐 Button 公开属性与加载态

**功能分支**：`20260323-button-props-loading`  
**创建时间**：2026-03-23  
**状态**：草稿  
**输入**：用户描述："继续补齐button 的props：新增 onClick 和 onClickCapture 事件，color 新增 danger 颜色，新增 htmlType，用于设置button type，但是不影响样式，补充 href 和target，也是透传给button，然后暴露ref，是button 的dom节点。IconButton 也同步更新。然后再补个loading 的props，开启的时候按钮前面会显示loading转圈的icon，如果是IconButton 则icon变成loading转圈的icon，然后按钮样式是disable状态，但是不需要diasble 的cursor样式。"
**语言要求**：本文件及同级 `/specs/` 文档正文必须使用简体中文；代码标识符、命令、文件路径、协议字段和第三方 API 名称可保留原文。

## 用户场景与测试（必填）

### 用户故事 1 - 以统一 API 接入按钮行为（优先级：P1）

作为消费组件库的应用开发者，我希望 `Button` 与 `IconButton` 能直接提供常见交互与语义属性，包括点击事件、按钮提交语义、原生链接语义以及可拿到真实根节点的 ref，这样我不需要再包一层自定义组件或自行补齐原生能力。

**为什么是这个优先级**：公开属性不完整会直接阻碍接入，开发者即使认可现有视觉样式，也无法把组件用于表单提交、事件捕获、真实链接跳转或命令式聚焦等实际场景。

**独立测试**：在不编写额外包装层的前提下，分别使用 `Button` 与 `IconButton` 配置 `onClick`、`onClickCapture`、`htmlType`、`href`、`target` 和 ref，并验证这些能力在常见交互场景下可直接使用且不改变现有视觉样式选择方式。

**验收场景**：

1. **假如** 开发者需要响应按钮点击或捕获阶段事件，**当** 其为 `Button` 或 `IconButton` 配置 `onClick` 与 `onClickCapture`，**那么** 对应事件会按公开契约触发，而不需要额外封装。
2. **假如** 开发者需要让按钮参与表单提交或重置，**当** 其配置 `htmlType`，**那么** 按钮语义会按配置生效，且视觉样式不会因该字段变化而改变。
3. **假如** 开发者需要把按钮渲染为真实链接，**当** 其配置 `href`，**那么** 组件会以 `<a>` 根节点渲染，并在保留既有视觉样式的同时提供原生链接激活行为。
4. **假如** 开发者需要在外部聚焦按钮或读取按钮节点，**当** 其获取 `Button` 或 `IconButton` 的 ref，**那么** ref 会指向实际渲染根节点。

### 用户故事 2 - 用危险态和加载态表达关键操作（优先级：P2）

作为设计师和应用开发者，我希望按钮能表达破坏性操作和进行中的操作：`color` 新增 `danger` 用于危险动作，`loading` 用于展示处理中状态，并在普通按钮与图标按钮中都有一致的视觉反馈。

**为什么是这个优先级**：危险态与加载态都是高频业务场景。缺少它们会迫使业务方私自改色或拼接临时 loading 图标，破坏组件库的一致性。

**独立测试**：分别查看 `Button` 与 `IconButton` 的默认、`danger`、`loading` 组合样例，确认普通按钮在 `loading` 时前置显示转圈图标，`IconButton` 在 `loading` 时以转圈图标替换原图标，且按钮整体表现为不可再次触发的禁用态视觉，但鼠标指针不切换为 disabled 样式。

**验收场景**：

1. **假如** 开发者需要呈现删除、移除等破坏性动作，**当** 其将按钮颜色设为 `danger`，**那么** 按钮会以明确区别于现有中性和强调色的危险态样式展示。
2. **假如** 普通 `Button` 进入 `loading`，**当** 用户查看按钮内容，**那么** 文本前会显示 loading 转圈图标，并保持按钮原有内容可理解。
3. **假如** `IconButton` 进入 `loading`，**当** 用户查看按钮内容，**那么** 原图标会被 loading 转圈图标替换，避免同一位置同时出现两套图标。
4. **假如** 任意按钮处于 `loading`，**当** 用户尝试再次触发该按钮，**那么** 组件会表现为禁用态，不再鼓励重复操作，同时鼠标指针不显示 disabled 专属样式。

### 用户故事 3 - 在预览与无障碍场景下保持一致（优先级：P3）

作为组件维护者、仅键盘用户和评审者，我希望新增属性与状态在 `Button`、`IconButton`、主要预览面和无障碍场景里保持一致，这样新增能力不会只在单一路径可用，或在状态切换时丢失可读性与可操作预期。

**为什么是这个优先级**：本次变更跨越公开 API、状态反馈和派生组件。如果一致性不足，后续使用者会在 Button 与 IconButton 之间得到分叉行为。

**独立测试**：检查预览中 `Button` / `IconButton` 的事件、`danger`、`loading`、ref、`htmlType`、`href`、`target` 代表性样例，并验证键盘聚焦、状态识别和辅助技术命名在新增状态下保持明确。

**验收场景**：

1. **假如** 评审者查看主要预览入口，**当** 其检查 `Button` 与 `IconButton` 的新增属性和状态样例，**那么** 能看到一致的公开能力面，而不是只有单一组件更新。
2. **假如** 用户仅通过键盘访问按钮，**当** 按钮处于默认、危险态或加载态，**那么** 焦点可见性和状态识别仍保持清晰。
3. **假如** 一个只有图标的按钮进入 `loading`，**当** 辅助技术读取该控件，**那么** 仍能获得可理解的按钮名称与当前状态，而不是只读出无意义图标。

### 边界情况

- 当按钮同时配置 `loading` 与点击事件时，系统如何避免重复触发并保持处理中状态可感知？
- 当按钮配置 `htmlType`、`href` 或 `target` 时，系统如何保证这些字段只影响根节点语义而不意外切换视觉样式？
- 当 `Button` 或 `IconButton` 处于 `loading` 时，系统如何在保留可读性的同时展示禁用态外观，但继续保留常规鼠标指针反馈？
- 当按钮已经处于 `disabled` 且再开启 `loading` 时，系统如何保证状态结果可预测，不出现冲突反馈？
- 当仅图标按钮切换到 `loading` 图标后，系统如何保证按钮名称、焦点可见性和危险态识别仍然明确？
- 当开发者通过 ref 操作按钮节点时，系统如何保证 `Button` 与 `IconButton` 暴露的一致性？

## 需求（必填）

### 功能需求

- **FR-001**：系统必须为 `Button` 与 `IconButton` 同步公开 `onClick` 与 `onClickCapture`，使消费方可直接接入冒泡阶段与捕获阶段的点击处理。
- **FR-002**：系统必须为 `Button` 与 `IconButton` 公开 `htmlType`，用于设置按钮语义类型，且该字段不得作为视觉样式选择条件。
- **FR-003**：系统必须为 `Button` 与 `IconButton` 公开 `href` 与 `target`；当存在 `href` 时，组件必须以真实 `<a>` 根节点渲染，并保持既有视觉样式契约。
- **FR-004**：系统必须为 `Button` 与 `IconButton` 暴露 ref，且 ref 指向实际渲染根节点。
- **FR-005**：系统必须在现有 `color` 能力上新增 `danger`，用于表达破坏性或高风险操作。
- **FR-006**：系统必须保证 `danger` 与现有按钮视觉体系共存，使用者无需切换到另一套组件即可表达危险动作。
- **FR-007**：系统必须为 `Button` 与 `IconButton` 新增 `loading` 状态字段，并保持两个入口的命名与行为一致。
- **FR-008**：当普通 `Button` 进入 `loading` 时，系统必须在按钮内容前展示 loading 转圈图标。
- **FR-009**：当 `IconButton` 进入 `loading` 时，系统必须用 loading 转圈图标替换当前图标，而不是叠加显示。
- **FR-010**：当按钮处于 `loading` 时，系统必须呈现与禁用态一致的视觉反馈，使用户感知该按钮当前不可重复触发。
- **FR-011**：当按钮处于 `loading` 时，系统不得切换为 disabled 专属鼠标指针样式，避免与用户要求冲突。
- **FR-012**：当按钮处于 `loading` 时，系统必须阻止重复激活，直到 `loading` 结束。
- **FR-013**：系统必须保证 `loading`、`danger`、`disabled` 与现有 `variant`、尺寸、形状能力的组合结果可预测，并在公开文档或预览中有代表性覆盖。
- **FR-014**：系统必须保证新增 `htmlType`、`href`、`target`、事件字段与 ref 不改变既有按钮的默认样式选择、颜色选择或尺寸选择逻辑。
- **FR-018**：系统必须保证 `target` 只有在 `href` 存在时才能使用；若单独传入 `target`，必须给出明确错误提示。
- **FR-015**：系统必须继续满足按钮无障碍要求，包括键盘聚焦、可访问名称、状态识别，以及仅图标按钮在 `loading` 与 `danger` 下的可理解性。
- **FR-016**：系统必须在 `website` 中提供 `Button` 与 `IconButton` 的代表性预览，覆盖默认、`danger`、`loading`、已禁用、仅图标、带文本、事件接入和 ref 使用场景。
- **FR-017**：系统必须明确说明 `loading` 与 `disabled` 的关系，使消费方能判断同时使用时的预期行为与视觉结果。

### 无障碍与 UI 契约（UI 工作必填）

- **用户角色 / 参与方**：消费组件库的应用开发者、评审视觉语义的设计师、组件维护者、仅键盘用户、屏幕阅读器用户
- **交互模型**：鼠标点击、捕获阶段点击处理、键盘聚焦与激活、表单提交语义、导航相关字段透传、命令式 ref 访问、普通按钮与图标按钮的 loading 切换
- **需要覆盖的状态**：默认、hover、focus-visible、active、disabled、loading、danger、danger + loading、仅图标、图标加文本、事件接入、ref 暴露、`htmlType`、`href`、`target`
- **主题 / token 影响**：复用现有按钮层级、危险态语义色、禁用态视觉反馈和 loading 图标风格；若现有 token 不足以表达 `danger` 或 loading 状态，需要明确补充或复用策略

### 关键实体（如功能涉及数据则填写）

- **按钮公开属性集**：描述 `Button` 与 `IconButton` 对外暴露的交互、语义、导航、状态和 ref 能力，包括 `onClick`、`onClickCapture`、`htmlType`、`href`、`target`、`loading` 与 `color="danger"`。
- **按钮加载态**：描述按钮进入处理中后的视觉、交互和图标替换规则，包括普通按钮前置 loading 图标、图标按钮图标替换、禁用态外观和重复触发限制。
- **按钮危险态**：描述破坏性操作使用的颜色语义，以及其在常规按钮与图标按钮中的一致表达方式。
- **按钮预览样例**：描述需要在预览面中展示的代表性组合，用于评审公开 API、一致性和边界状态。

## 依赖与假设

### 依赖

- 本功能依赖现有 Button / IconButton 的视觉变体、尺寸、形状和无障碍基线可以继续复用。
- 本功能依赖组件预览入口可展示新增属性和状态，供设计师与开发者共同评审。
- 本功能依赖现有按钮体系已具备可复用的禁用态反馈和 loading 图标表达方式，或允许在同一语义体系内补齐。

### 假设

- 本期 `danger` 作为 `color` 的新增可选值引入，不额外新增独立的危险态组件。
- 本期 `htmlType` 仅用于表达按钮语义类型，不承担任何视觉样式切换职责。
- 本期 `href` 与 `target` 作为公开属性补齐，重点是让消费方可按统一契约传入，并直接获得原生链接根节点。
- 本期 `loading` 的主要目标是阻止重复触发并让处理中状态可见，因此其视觉反馈应接近禁用态，但鼠标指针继续保持非 disabled 样式。
- 本期 `Button` 与 `IconButton` 的新增能力范围保持一致，除 loading 图标呈现方式不同外，不引入分叉的命名或接入方式。

## 成功标准（必填）

### 可度量结果

- **SC-001**：100% 的代表性接入场景可直接通过 `Button` 或 `IconButton` 配置点击事件、`htmlType`、`href`、`target` 和 ref 完成，不再需要额外包装层，其中带 `href` 的场景都能获得真实链接激活行为。
- **SC-002**：100% 的 `danger` 代表性样例都能被评审者明确识别为破坏性动作，且与现有中性和强调色区分清晰。
- **SC-003**：100% 的 `loading` 代表性样例都能在 1 秒内被用户识别为“处理中且不可重复触发”的按钮状态。
- **SC-004**：100% 的普通 `Button` loading 样例显示前置 loading 图标，100% 的 `IconButton` loading 样例以 loading 图标替换原图标。
- **SC-005**：100% 的 `loading` 样例都表现为禁用态视觉，但不会出现 disabled 专属鼠标指针。
- **SC-006**：`Button` 与 `IconButton` 的新增属性和状态在预览面覆盖率达到 100%，至少包含默认、`danger`、`loading`、已禁用、仅图标和图标加文本场景。
- **SC-007**：100% 的仅图标按钮在默认、`danger` 或 `loading` 状态下都具备可访问名称，并通过键盘聚焦检查。
